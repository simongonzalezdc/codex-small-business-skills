# SEO Notes

BLUF: GitHub discovery depends mostly on repository name, description, topics,
README clarity, stars, inbound links, and whether the repo is public and
indexable.

## Primary Search Targets

- Codex small business skills
- Codex skills for small business
- Claude small business plugin Codex port
- Claude Code skills for SMB workflows
- AI agents for small business workflows
- open-source small business AI automation
- cash flow invoice CRM customer support AI skills

## On-Page Checklist

- [x] Keyword-rich H1: `Codex Small Business Skills`
- [x] Short value proposition in first paragraph
- [x] Apache-2.0 license badge and file
- [x] Upstream source and attribution
- [x] Clear install command
- [x] Included skill list
- [x] Use cases in plain language
- [x] Professional-advice disclaimer
- [x] Recommended GitHub topics
- [x] Submission kit for off-GitHub promotion

```text
codex, codex-skills, claude-code, claude-plugin, ai-agents, small-business, smb, workflow-automation, business-automation, crm, cash-flow, apache-2-0, open-source
```

## Search Engine Notes

After the public URL exists:

1. Make sure the repository is public.
2. Link to it from at least one indexed website or profile.
3. Use Google Search Console URL Inspection only if the URL is under a verified
   property you control.
4. Keep the README useful and specific; thin launch pages tend to underperform.
